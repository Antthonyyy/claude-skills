# /creative — Data-Driven Ad Creative Generator

Генерация рекламных креативов (текст + визуальное ТЗ) на основе реальных данных проекта: конкуренты, метрики кабинета, боли ЦА. Не шаблоны — готовые к запуску материалы.

## Вызов

```
/creative имя_проекта              → все этапы воронки (cold + warm + retarget)
/creative имя_проекта cold         → только холодная аудитория
/creative имя_проекта warm         → тёплая
/creative имя_проекта retarget     → ретаргетинг
```

Аргумент `имя_проекта` = имя папки в `projects/` (например `wnw`, `rawform`).

---

## Фаза 0: Валидация

1. Парсить аргументы: `{project}` и `{stage}` (default: `all`)
2. Проверить что `projects/{project}/CLAUDE.md` существует — если нет, сообщить и остановиться
3. Прочитать `projects/{project}/CLAUDE.md` — извлечь:
   - Ниша и продукт
   - ЦА (кто, гео, язык, сегменты)
   - Воронка (куда ведёт реклама: DM, WhatsApp, лендинг, форма)
   - Тон текстов и язык (русский / English)
   - Конкуренты (если указаны)
   - CTA механика (trigger word в директ, WhatsApp, ссылка)

## Фаза 1: Сбор данных

**Принцип: сначала читаем то, что уже есть. Скрипты запускаем только если данных нет.**

### 1a. Данные проекта (читать через glob + read)

```
projects/{project}/CLAUDE.md                        → контекст проекта
projects/{project}/tasks/research_*.md              → последний research (боли ЦА, конкуренты)
projects/{project}/tasks/competitive_analysis_*     → анализ объявлений конкурентов
projects/{project}/tasks/creative_brief_*           → готовый креативный бриф
projects/{project}/tasks/strategy_*.md              → стратегия с сегментами ЦА
projects/{project}/tasks/analysis_*.md              → метрики кампаний (CTR, CPL — что работает)
projects/{project}/tasks/creative_analysis_*        → анализ существующих креативов
projects/{project}/tasks/copy_ads_*.md              → прошлые тексты (не повторяться)
projects/{project}/tasks/decisions_and_tz_*         → прошлые ТЗ на креативы
```

Из каждого файла брать только последний по дате (самый свежий).

### 1b. База знаний (читать ВСЕГДА)

```
knowledge/frameworks/copywriting.md              → формулы PAS/AIDA/4U с примерами
knowledge/advertising/creatives.md               → форматы, размеры, первые 3 сек, чеклист
knowledge/frameworks/funnels.md                  → лестница Ханта (5 уровней осознанности)
knowledge/advertising/meta-ads-tactics-2026.md   → тактики 2026, что работает сейчас
knowledge/frameworks/audience.md                 → JTBD, аватар, сегментация
```

Проектная база знаний (если есть):
```
projects/{project}/*База_знаний*.md
projects/{project}/*knowledge*.md
```

### 1c. Если данных нет — запустить скрипт

**Условие:** нет ни одного файла `research_*.md` и `competitive_analysis_*` (или все старше 30 дней).

```bash
python scripts/ad_pipeline.py --query "{поисковый запрос из контекста проекта}" --country {код_страны} --project {project}
```

Поисковый запрос формируется из ниши и ключевых слов CLAUDE.md проекта:
- WNW → `"женский нетворкинг"` / `"women networking europe"`
- Rawform → `"villa furniture bali"` / `"custom furniture bali"`

Если `ad_pipeline.py` падает — сообщить Антону и работать с тем, что есть.

## Фаза 2: Grounding — сводка данных

Перед генерацией текстов — **собрать контекстный блок** из прочитанных файлов. Это "земля" под ногами — без неё не писать.

### Что извлечь:

**Из competitive_analysis / creative_brief:**
- Что делают конкуренты (форматы, хуки, CTA)
- Гэпы рынка — чего НЕ делает никто (= наши возможности)
- Референсы (конкретные объявления, которые работают)

**Из analysis / метрик кабинета:**
- Кампании с CTR > 1.5% — что в них было (хук, формат, аудитория)
- Кампании на паузе / CTR < 0.8% — что НЕ работало (избегать этих подходов)
- Текущая частота, CPM, CPL — для понимания ситуации

**Из research / YouTube:**
- Боли ЦА — **дословные цитаты** из комментариев
- Триггеры и возражения
- Язык, которым ЦА описывает проблему (использовать в хуках)

**Из базы знаний проекта:**
- Social proof — реальные цифры (количество участников, стран, лет работы, проектов)
- Продуктовые детали — цены, даты, города, форматы
- Отзывы клиентов (если есть)

### Правило заземления

**Каждый** текст объявления и каждое визуальное ТЗ должны содержать строку:

```
**Заземление:** {откуда взята идея — конкретный источник}
```

Примеры:
- `Заземление: Боль #2 из YouTube research ("Правильные связи = деньги, но нетворкинг не работает"), гэп рынка (нет видео-крео у конкурентов)`
- `Заземление: Кампания "Мюнхен" CTR 3.16% — хук-вопрос + живое фото работают. Берём формат, меняем город.`
- `Заземление: Конкурент X использует showcase-формат с сеткой фото → мы добавляем производственные кадры (наш дифференциатор)`

Если текст нельзя заземлить — он generic и подлежит переписке.

## Фаза 3: Генерация текстов

### Формулы по этапам воронки

| Этап | Формула | Длина | Количество хуков | Количество вариантов | Источник боли/желания |
|---|---|---|---|---|---|
| cold | PAS или AIDA | 100-250 слов | 5+ | 2-3 | Боли из research, цитаты ЦА |
| warm | Social proof / Результат | 60-120 слов | 3+ | 2-3 | Метрики наших кампаний + social proof |
| retarget | ODC (Offer-Deadline-CTA) | 30-60 слов | 2+ | 2 | Продукт и оффер из CLAUDE.md |

### Формулы (справка)

**PAS — Problem → Agitate → Solution:**
- P: Описать ситуацию языком клиента. Без диагноза.
- A: Что теряет клиент? Что будет, если не решить?
- S: Конкретный выход. Одно действие.

**AIDA — Attention → Interest → Desire → Action:**
- A: Хук — останавливает скролл
- I: Показать понимание ситуации
- D: Картина результата
- A: Один чёткий CTA

**4U — для хуков:**
- Useful (что получит)
- Unique (чем отличается)
- Ultra-specific (цифры, даты)
- Urgent (почему сейчас)

**ODC — для ретаргетинга:**
- Offer (конкретный оффер)
- Deadline (срок)
- Call-to-action (одно действие)

### Форматы хуков

Типы хуков для генерации (чередовать):
- **Провокационный вопрос:** `Строишь бизнес в Германии, но нетворкинг идёт в никуда?`
- **Шокирующий факт:** `85% успеха — это люди. А не твои навыки.`
- **Обращение к ситуации:** `15 мест. Бизнес-ланч для предпринимательниц в Гамбурге.`
- **Результат наоборот:** `Почему умные женщины застревают — даже с опытом и деньгами.`
- **Цифра + боль:** `3 года в Европе. 0 деловых контактов.`
- **Прямое предложение:** `Furnishing multiple villas at once? One team handles everything.`

### Формат каждого варианта

```markdown
### {STAGE} — Вариант {A/B/C} ({формула}: {PAS/AIDA/ODC})

**Хук:**
{первые 2 строки — отдельно, это то что видно до "ещё"}

**Текст:**
{полный текст}

**CTA:**
{конкретный призыв — одна строка с механикой: DM trigger / WhatsApp / ссылка}

**Headline (под картинкой):** {если Meta Ads — заголовок объявления}
**Description:** {описание}
**CTA кнопка:** {Send Message / Learn More / Sign Up / Shop Now}

**Заземление:** {источник идеи}
```

Каждый вариант — **ГОТОВЫЙ текст**. Ни одного `[вставить город]` или `[имя клиента]`. Все данные — из контекста проекта.

## Фаза 4: Визуальные ТЗ

Для каждого этапа воронки — 1-2 визуальных ТЗ.

### Выбор формата

На основе анализа конкурентов и метрик:
- Конкуренты не используют видео → **приоритет видео** (гэп рынка)
- У нас статика с CTR > 2% → **статика** (работает — масштабируем)
- У нас видео с CTR < 1% → **статика** или **другой тип видео**
- Ниша B2B → **статика 1:1** или **короткое видео 15 сек**
- Ниша B2C с Instagram → **Reels 9:16**

### ТЗ для видео (Reels / Stories / Feed Video)

```markdown

**Длительность:** {15-60 сек}

**Shot list:**

| 10-20 сек | {что снять} | "{текст}" | "{текст}" |
```

## Фаза 5: Проверки (3 гейта)

### Гейт 1 — Anti-AI Sniff Test

Прогнать КАЖДЫЙ текст через чек-лист:

**Запрещённые фразы (удалить/переписать):**
- "В современном мире", "На сегодняшний день"
- "Давайте разберёмся", "Хочу поделиться"
- "Важно отметить", "Стоит отметить"
- "Безусловно", "Несомненно", "Очевидно"
- "Данный" (→ "этот"), "Является" (→ "это")
- "В рамках", "Осуществлять", "Оптимизировать", "Масштабировать"
- "Трансформация", "Синергия", "Экосистема"
- "Ключевой момент", "Важный аспект"
- "Комплексный подход", "Уникальное предложение" (без конкретики)
- Фейковая эмпатия: "Мы все через это проходили", "И это нормально"
- Мотивационные цитаты и лозунги

**Запрещённые структуры:**
- Зеркальное начало (пересказ темы первым предложением)
- Проповедь из трёх пунктов (вступление → 3 пункта → вывод)
- Резюме в конце ("Итак, мы рассмотрели...")
- "С одной стороны... с другой стороны" — иметь позицию
- Нарастающий восторг: "Первое... Второе... И самое важное!!!"

**7-Point Test:**
1. Это мог написать кто угодно для любого бизнеса? → Добавить конкретику проекта
2. Есть запрещённые фразы? → Найти и заменить
3. Начинается с пересказа темы? → Переписать начало
4. Заканчивается резюме? → Отрезать последний абзац
5. Каждое утверждение подкреплено числом из данных проекта? → Если нет — добавить или убрать
6. Конкурент может забрать этот текст себе? → Если да — слишком generic, переписать
7. Прочитай вслух — звучит как реклама или как человек? → Переписать что звучит как брошюра

### Гейт 2 — Meta Ads Policy

**Запрещено:**
- Личные характеристики аудитории в лоб: "Ты иммигрантка", "Если ты одинока", "Для тех у кого нет друзей" → переформулировать через ситуацию: "Переехала в новый город?"
- Гарантированные результаты: "Ты точно найдёшь", "Гарантируем" → убрать
- Финансовые обещания без дисклеймера: "Заработай 10K" → убрать или смягчить
- Дискриминация по возрасту, полу, религии
- "Бесплатно" в заголовке без условий
- До/после в изображениях
- Упоминание болезней, веса, внешности как проблемы

**Жёлтая зона (переформулировать):**
- "Ты не умеешь нетворкиться" → "Нетворкинг кажется чужим?"
- Эмоциональное давление: чрезмерный страх, стыд, одиночество как главный мотив

Если найдено нарушение — **исправить** и отметить что изменено.

### Гейт 3 — Тон проекта

Сверить с `projects/{project}/CLAUDE.md`:
- **Язык** соответствует (RU / EN / UA)
- **Тон** соответствует (тёплый-профессиональный / деловой-конкретный / raw-authentic)
- **Обращение** правильное ("ты" / "вы" / формальное)
- **Эмодзи** умеренно или отсутствуют (по проекту)

## Бенчмарки для оценки креативов (справка)

| Метрика | Хорошо | Проблема | Критично |
|---|---|---|---|
| CTR (link) | > 1.5% | 0.8-1.5% | < 0.8% |
| CPM | < $8 | $8-15 | > $15 |
| CPC | < $0.40 | $0.40-0.80 | > $0.80 |
| Frequency | < 2.5 | 2.5-3.5 | > 3.5 |

**Связь метрика → креатив:**
| Метрика | Проблема в креативе | Что менять |
|---|---|---|
| CTR < 0.8% | Слабый хук | Первые 3 сек видео / первая строка текста |
| CTR ок + мало конверсий | Слабый CTA | CTA кнопка / финальный кадр |
| CTR падает через 2 нед. | Выгорание | Новый визуал, тот же подход |
| Высокая частота + низкий CTR | Аудитория устала | Смена креатива или расширение аудитории |
