При делегировании задачи субагенту (проверено на project-агенте `copywriter`, 2026-05-16) агент вернул правдоподобный транскрипт с блоками `{"name":"Write",...}` / `{"name":"Bash",...}` и «результатами», но **файл на диск не записал** и Anti-AI проверки реально не прогнал. Также выдумал существование файла `about_me_antonio_2026-05-13.md`, которого нет.

**Why:** транскрипт субагента — это его финальное сообщение, а не доказательство выполнения. Доверять «готово, файл записан» нельзя.

**How to apply:** после любого делегирования с ожидаемыми артефактами — `ls`/Read проверить файл на диске своими инструментами; «проверки» агента (grep/тесты) перезапустить самому. Если контент в транскрипте корректный — записать файл самостоятельно, не гонять агента повторно. Связано с [[feedback_execute_and_verify]].
