Когда Антон даёт задачу формата «проанализируй / спроектируй / предложи roadmap / собери ТЗ», даже если в задаче упоминается конкретная фича к реализации (например «добавить add-admin»), — выдавать **только предложение** (markdown в `clients/имя/tasks/`). Никаких правок прод-кода, миграций, деплоев, скриптов автоматизации в этой же сессии.

**Why:** Антон 2026-06-05 явно остановил оркестратор сообщением «Без меня ничего не делай. Сейчас только предлагаешь» в середине задачи аудита WNW CRM, где упоминалась новая фича add-admin. Контекст: CRM-репо — production Татьяны, любые правки нужно проводить через явное согласование scope и тайминга.

**How to apply:**

- В задачах «аудит / план / roadmap / спецификация» — фаза EXECUTE оркестратора **останавливается** на собранном артефакте.
- Заранее в плане формулировать «следующий шаг — твоё решение по X + green light».
- Если задача формулируется как «реализуй» — это другой кейс, продолжаем как обычно. Триггер этого правила — слова «проанализируй», «спроектируй», «предложи», «что можно улучшить», «соберём ТЗ».
- Связано: [[feedback_dont_drift_diagnose_to_deploy]] — про дрейф из диагностики в правки кода.
